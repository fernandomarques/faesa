JavaScript - Erros Comuns

  • Usar o operador de atribuição ao invés do de igualdade ==
  • '10' == 10 → true
  • '10' === 10 → false
  • Switch usa comparação restrita (===)
  • 0.2 + 0.1 não é = 0.3 (floats...)
  • Quebrar uma string em linhas
  • var x = "Hello \

JavaScript - Boas Práticas

  • Evite o uso de variáveis globais (duh!)
  • Variáveis locais são declaradas com o var
  • Declare variáveis no início do script
  • E inicialize os valores!
  • Evite declarar objetos Number,String ou Boolean pois eles atrazam a execução
  • var a = new Number(1);
  • var x = new String('Fernando');
var y = new String('Fernando');
x == y // falso pois não podemos comparar objetos

JavaScript - Boas Práticas

  • Use {} instead of new Object()
  • Use "" instead of new String()
  • Use 0 instead of new Number()
  • Use false instead of new Boolean()
  • Use [] instead of new Array()
  • Use /()/ instead of new RegExp()
  • Use function (){} instead of new Function()

JavaScript - Boas Práticas

  • Sempre utilize ===
  • 0 == "";        // true
1 == "1";       // true
1 == true;      // true

0 === "";       // false
1 === "1";      // false
1 === true;     // false
