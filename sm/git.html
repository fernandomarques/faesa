Versionamento de código e Git

Created by Fernando Marques

O que é controle de Versão?

  • Sistemas de Controle de Versão são uma categoria de software que ajuda a uma equipe a controlar mudanças que acontecam no código fonte ao longo do tempo
  • São guardadas todas as modificações que ocorrem no código. Se um erro acontecer, podemos comparar com versões antigas para encontrar o problema
  • Controle de Versão é uma forma de proteger o código fonte

Mas é mesmo necessário?

  • Em empresas é comum que mais de um desenvolvedor trabalhe nos mesmos arquivos fontes, mesmo que um esteja criando uma funcionalidade nova e o outro corrigindo um bug
  • Evita conflitos, caso duas pessoas mudem o mesmo código
  • Caso o conflito ocorra, ele deve ser resolvido

Vantagens

  • Funciona como um backup. Não só do código atual, mas de todo código que já foi escrito!
  • Guarda informação sobre tudo que já foi feito, por quem e quando
  • Branching e Merging (próximo slide)
  • Rastreabilidade, permite saber por que o código foi modificado

Branching e Merging

  • Todo Software precisa evoluir...
  • Podemos criar "branches" de novas funcionalidades
  • Enquanto a funcionalidade é criada, e ainda não está estável, podem surgir bugs...
  • O bug é corrigido e a branch continua
  • Ao terminar, fazemos o merge

Git - Comandos Básicos

  • git init
  • git status
  • git add "file" ou .
  • git commit -m "Motivo do commit"
  • git pull
  • git push
  • git clone
  • arquivo .gitignore

Praticando, local

  1. Criar um novo repositório git init
  2. Criar um arquivo .txt e escrever algo
  3. Adicionar o arquivo git add .
  4. Comitar o arquio git add -m "mensagem"
  5. Modificar o arquivo
  6. Comitar novamente!
  • Atualmente é o sistema de versionamento mais utilizado
  • Foi criado por Linus Torvalds
  • É um sistema open source
  • Tem arquitetura distribuída
  • Ao contrário de outros sistemas de versionamentos que já foram populares (svn e cvs). Cada desenvolvedor tem uma cópia do código
  • Além de ser distribuído, foi desenvolvido levando em conta performance, segurança e flexibilidade

Git - Performance

  • A performance é considerada melhor do que das outras alternativas
  • Leva em consideração atributos comuns em um arquivo de código
  • Não é enganado quando um arquivo muda de nome, considerando mais o conteúdo do arquivo
  • O fato de ser distribuído também melhora a performance
  • Como possui cópia local, não é necessário acessar servidor para trocar de branchs e versões

Git - Segurança

  • Git considera integridade com alta prioridade
  • O conteúdo dos arquivos bem como a relação entre arquivos, pastas, versões, tags e commits são protegidos
  • A proteção acontece usando uma chave criptográfica, no caso, SHA1
  • Esse mecanismo garante que nenhuma alteração 'secreta' aconteça
  • Caso um dos itens seja modificado, a chave mudaria também!

Git - Flexibilidade

  • Suporta vários workflows não lineares de desenvolvimento
  • É compatível com muitos sistemas e protocolos
  • Branching e Tagging garantem grande flexibilidade

Git

  • Quando comparado aos concorrentes se sai melhor
  • É amplamente utilizado na web
  • É um projeto open source de qualidade
  • Existem diversos tutoriais para git, inclusive em português!

Git - Criticas

  • Algumas pessoas dizem que é difícil a aprendizagem
  • Quem está acostumado com versionamento centralizado pode demorar a entender o versionamento distribuído. Exemplo: EU
