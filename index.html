Motivação de Usar Classes

  • Agrupar dados semelhantes
  • Exemplo: Calcular média de 03 notas de 20 alunos
  • Encapsulamento e Abstração
  • Herança
  • Polimorfismo

Mas em C já podemos fazer isso!


struct Notas {
	float nota1;
	float nota2;
	float nota3;
};
int main(){
	struct Notas aluno1;
	aluno1.nota1 = 10;
}

Por outro lado, em C++


class Notas {
	public:
		float nota1;
		float nota2;
		float nota3;
};
int main() {
	Notas aluno1;
	aluno1.nota1 = 10;
}

Adicionando métodos e implementando


class Notas {
	private:
		float Nota1;
		float Nota2;
		float Nota3;
	public:
		int media();
}
int Notas::media(){
	return (Nota1+Nota2+Nota3)/3
}

Controle de Acesso

  • Ao declararmos um atributo ou método de uma classe também definimos seu escopo
  • Public: É público e pode ser acessado por qualquer um que veja a classe
  • Protected: Só pode ser visto por classes filhas. Classes que herdam
  • Private: Os métodos e variáveis só podem ser acessados pela própria classe

Mas de que adianta ter um método ou atributo privado?

  • Getter/Setter
    • Atributos são protegidos e só podem ser acessados usando funções específicas
    • Getter = Pega o valor do atributo
    • Setter = Altera o valor do atributo
  • Algumas funções são usadas internamente e não poderiam ser chamadas por outras classes

class Conta
{
  int numero;     // São atributos
  string nome;    // privados por
  float saldo;    // default
public:
  void deposita(float valor);
  void consulta();
  int saque(float valor);
};
int Conta::saque(float valor){
	if (saldo < valor)
		return 0;
	else{
		saldo = saldo - valor;
		return 1;
	}
}

Classes e Objetos

  • As classes funcionam como uma máscara do que um objeto deve ter
  • Classes ainda não algo 'abstrato', pense em um super-herói. Pode ser Wolverine, Ironman, Capitão América...
  • Quando instanciamos a classe, temos algo mais 'concreto': um objeto
Classes e Objetos

Construtores e Destrutores

  • São métodos especias que são chamados de forma implícita
  • Quando um objeto de uma classe é criado, esse método é chamado
  • Assim, devemos nesse método fazer todas as inicializações de variáveis
  • Da mesma forma, o destrutor é chamado sempre que um objeto deixa de existir
Construtores e Destrutores 

#include 
class Notas {
	float Nota1, Nota2, Nota3;
	public:
		Notas();
		~Notas();
};
Notas::Notas(){
	std::cout << "Eu existo! :)\n";
}
Notas::~Notas(){
	std::cout << "Não existo mais :(\n";
}
int main(){
	Notas nota;
}

Boas Práticas

  • É recomendado que as declarações sejam feitas no arquivo de cabeçalho .h
  • Já a implementação dos métodos é feita no arquivo .cpp, que deve incluir o cabeçalho
  • Assim, podemos distribuir os cabeçalhos e os binários das implementações

Exercícios

  • Crie uma classe Notas
  • Deve Conter, como atributos privados, string Nome e float Nota1, float Nota2 e float Nota3;
  • Implemente os métodos, como public: media e os getters e setters
  • Para os getters basta retornar os valores das variáveis
  • Os setters devem ler do terminal. Para as notas, garanta que elas não sejam menores que 0 ou maiores que 10
