## Divisão e Conquista - Muitos algoritmos úteis são __recursivos__, chamando a si mesmo para resolver um problema - O problema é quebrado em vários subproblemas semelhantes mas de menor tamanho - O paradigma de divisão e conquista envolve três passos: 1. _Divisão_ do problema em subproblemas, resolvendo-os recursivamente 1. _Conquista_ os subproblemas 1. _Combinação_ as soluções dadas aos subproblemas na solução do problema original
## Ordenação por intercalação - __Divisão__ divide a sequência de n elementos que deve ser ordenada em duas subsequências de n/2 elementos cada - __Conquista__ Ordena as duas subsequências recursivamente, utilizando a ordenação por intercalação - __Combinação__ Intercala as duas subsequências para produzir a resposta ordenada 

def merge(left, right):
	if not len(left) or not len(right):
	return left or right
	result = []
	i, j = 0, 0
	while (len(result) < len(left) + len(right)):
		if left[i] < right[j]:
			result.append(left[i])
			i+= 1
		else:
			result.append(right[j])
			j+= 1
		if i == len(left) or j == len(right):
			result.extend(left[i:] or right[j:])
			break 

	return result			

def mergesort(list):
	if len(list) < 2:
		return list
	middle = len(list)/2
	left = mergesort(list[:middle])
	right = mergesort(list[middle:])
	return merge(left, right)

## Relações de Recorrência - As recorrências andam juntas dos algoritmos de divisão e conquista - São equações de igualdade ou desigualdade que define uma função em termos de seu valor para entradas menores $$ T(n) = \Theta(1), se \, n=1 $$ $$ 2T(n/2) + \Theta(n), se\, n >1 $$ - Nesse casso a solução é $ T(n) = \Theta(n\,lgn) $
## Relações de Recorrência - As recorrências podem tomar muitas formas, podendo dividir em tamanhos desiguais como 2/3 e 1/3 - Tal algoritmo daria origem à recorrência $ T(n) = T(2n/3) + T(n/3) + \Theta(n) $ - Os subproblemas não estão necessariamente restritos a ser uma fração constante do tamanho do problema original - Uma versão recursiva da busca linear criaria um subproblema contendo um elemento a menos que o original - $T(n) = T(n-1) + \Theta(1) $
## Relações de recorrência - Veremos três métodos para resolver recorrências 1. __método de substituição__ arriscamos um palpite para um limite e então usamos indução matemática para provar o palpite 1. __método da árvore de recursão__ converte a recorrência em uma árvore cujos nós representam os custos envolvidos em vários níveis de recursão 1. __método metre__ dá limite a recorrências da forma $T(n) = aT(n/b) + f(n) $
## Relações de recorrência - Veremos recorrência que não são igualdades, porém, desigualdades como $T(n) \le 2T(n/2) + \Theta(n) $ - Como a recorrência declara um limite superior, expressaremos sua solução usando a notação O em vez de Θ - Também poderíamos utilizar a notação Ω para uma recorência de limite inferior - Também ignoraremos o fato que para, por exemplo, valores ímpares o merge não terá divisão igual - A relação do merge pode ser enunciada como $ T(n) = 2T(n/2) + \Theta(n) $

## Subarranjo máximo recursivo - Se dividirmos o arranjo em dois, cada subarranjo pode ser encontrado em: - Inteiramente no subarranjo A[low..mid] - Inteiramente no subarranjo A[mid+1.. high] - Cruzando o onto médio de modo que $low \le i \le mid \le j \le high $ 

	def find_max_crossing_subarray(A, low, mid, high) : 
		max_left, max_right = mid, mid+1
		left_sum = A[mid]
		sum = 0
		for i in range(mid,low-1,-1):
			sum = sum + A[i]
			if sum > left_sum:
				left_sum = sum
				max_left = i
		right_sum = A[mid+1]
		sum = 0
		for i in range(mid+1,high+1):
			sum = sum + A[i]
			if sum > right_sum:
				right_sum = sum
				max_right = i
		return (max_left, max_right, left_sum+right_sum)
	

	def find_max_subarray(A,low=0,high=None):
		if high is None:
			high = len(A)-1
		if high == low:
			return (low,high,A[low])
		mid = (low+high)//2
		left_low,left_high,left_sum    = find_max_subarray(A,low,mid)
		right_low,right_high,right_sum = find_max_subarray(A,mid+1,high)
		cross_low, cross_high,cross_sum = find_max_crossing_subarray(A,low,mid,high)
		if left_sum >= right_sum and left_sum >= cross_sum:
			return left_low, left_high, left_sum
		elif right_sum > cross_sum:
			return right_low, right_high, right_sum
		else:
			return cross_low, cross_high, cross_sum

## Pisos e tetos - Para qualquer número real X denotamos o maior inteiro menor ou igual a x por $\left\lfloor x \right\rfloor $, piso de x, e o maior número por $ \left\lceil x \right\rceil $, teto de x - $ x-1 < \left\lfloor x \right\rfloor \le x \le \left\lceil x \right\rceil < x+1 $ - Então, para qualquer número inteiro teremos - $\left\lfloor x/2 \right\rfloor + \left\lceil x/2 \right\rceil = x $
## Logaritmos - $log_c(ab) = log_ca + log_cb$ - $log_ba^n = n log_ba $ - $log_b(1/a) = -log_ba$ - $log_ba = \frac{1}{log_ab} $ - $log_ba = \frac{log_ca}{log_cb} $
- Vamos determinar limites superiores e inferiores a - $ T(n) = 2T(n/2) + n $ - Arriscamos o palpite $ T(n) = O(n\,lg\,n) $ - Devemos provar que $T(n) \le cn\,lg\,n $ para uma constante c > 0 - $ T(n) \le 2(c\,\left\lfloor n/2 \right\rfloor\,lg(\left\lfloor n/2 \right\rfloor)) + n $ - $ \le cn\, lg(n/2) + n $ - $ = cn\,lgn - cn\,lg2 + n $ - $ = cn\,lgn -cn + n $ - $ \le cn\,lg\,n $ - A última epata é válida para $ c \ge 1 $
## Condição de Contorno - Depois da prova por indução devemos mostrar que a condição é válida para a condição de contorno - $T(1) = 1 $ - Então $ T(1) \le c1\,lg\,1 = 0 $ - O que está em desacordo com $T(1) = 1$ - A nossa notação assintótica é valida para $ n \ge n_0 $ - Temos $T(2) = 4$ e $T(3) = 5$ - $ T(2) \le c2\,lg\,2$ e $T(3) \le c3\,lg\,3 $ - Qualquer valor $ c \ge 2 $ é suficiente para os casos base - Normalmente o caso de contorno é mais simples 😀
## O Palpite - Arriscar um palpite exige eficiência e criatividade - A árvore de recursão pode gerar bons palpites - Relações semelhantes podem ter palpites similares - $ T(n) = 2T(\left\lfloor n/2 \right\rfloor\ + 17) +n $ - O 17 parece dificultar o cálculo, mas para valores grandes de n, n/2 e n/2 + 17 cortam n quase pela metade - Assim $n\, lg\,n $ é um bom palpite - Outra forma é colocar limites frouxos como $\Omega(n)$ e $O(n^2)$ e tentar ir diminuindo até convergir a solução correta
